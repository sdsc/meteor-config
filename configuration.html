Configuring the pi to work with the network sandbox server

  1. Make sure the sandbox server is connected to wall port C
  2. Make sure meteor is connected to wall port B
  3. sudo nano /etc/network/interfaces
  4. Make sure at some point in the file it says this:
    5.        iface eth0 inet static
       address 10.0.0.X(Make sure the ip is different for every pi)
       netmask 255.0.0.0
       gateway 10.0.0.1
       broadcast 10.255.255.255
  5. Test whether this works with "ping 8.8.8.8"

Configuring the pi to work with the google public dns server

  1. sudo nano /etc/resolv.conf
  2. Under the initial line, make sure it says "nameserver 8.8.8.8"
